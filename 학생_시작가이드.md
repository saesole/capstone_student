각 메시지 형식은 노드 상단 주석과 `STRUCTURE.md` 참조.

## 3. 개발 순서 (권장)
1. **환경 기동**: `docker compose up --build` → noVNC(localhost:8080) 또는 가이드대로
2. **빌드**: 컨테이너에서 `cd ~/ros2_ws && colcon build && source install/setup.bash`
3. **Node A부터** 구현 → 단독 확인:
   ```bash
   ros2 run nd1_capstone node_a_llm
   # 다른 터미널
   ros2 topic pub --once /llm_command std_msgs/String '{data: "A구역 박스를 B구역으로 옮겨줘"}'
   ros2 topic echo /mission     # 올바른 JSON이 나오나?
   ```
4. **Coordinator** 구현 → `sim_mode:=true`로 전체 흐름:
   ```bash
   ros2 launch nd1_capstone bringup.launch.py sim_mode:=true
   # 다른 터미널
   python3 src/nd1_capstone/scripts/smoke_test.py
   ```
   → `SMOKE TEST: PASS` 가 1차 목표 (Node B/C는 sim에서 자동 성공 처리)
5. **Node B·C** 구현 → 실연동(`sim_mode:=false`)으로 실제 주행/파지
   (절차: `실연동_검증_절차.md`, 박스/월드: `기본월드_박스세팅.md`)

## 5. 막힐 때 (트러블슈팅)
- 토픽 흐름: `ros2 topic echo /robot_status` 로 어느 단계에서 멈추는지 확인
- 환경/RTF/실연동: `실연동_검증_절차.md`(T1~T11), `WSLg_렌더_가이드.md`

  - ★ 파지 타깃 y-offset ≥ 0.20 (y=0 특이점 회피)
