# 기본 월드(TurtleBot4) 재활용 — 박스 세팅 & 실연동 테스트

> 새 월드를 만들지 않고 TurtleBot4 ignition 기본 월드를 그대로 쓰는 절차.
> 새로 추가된 자산: `models/box1.sdf`, `launch/spawn_boxes.launch.py`.

## 0. 핵심 개념
- 파지 방식 = **텔레포트**(박스 삭제→재생성). 물리 팔 없이도 데모 성립.
- Node C는 `world_name`(=실행 중 월드 이름)으로 `/world/<world>/remove`·`/create` 서비스를 호출.
- 따라서 **월드명·박스명이 실제 시뮬과 일치**해야 함.

## 1. 시뮬 + Nav2 기동 (기본 월드)
```bash
ros2 launch turtlebot4_ignition_bringup turtlebot4_ignition.launch.py    # 터미널 1
ros2 launch nav2_bringup navigation_launch.py                            # 터미널 2
```

## 2. ★ 실제 월드명 확인 (가장 중요)
```bash
ign topic -l | grep -m1 world          # 예: /world/warehouse/clock → 월드명 = warehouse
# 또는
ign service -l | grep create           # /world/<월드명>/create 확인
ign model --list                        # 존재 모델 확인
```
TurtleBot4 기본 월드는 보통 `warehouse`. 확인된 이름을 아래 `<월드명>`에 사용.

## 3. 박스 스폰 (A구역에 box1 1개)
```bash
cd ~/ros2_ws && colcon build && source install/setup.bash
ros2 launch nd1_capstone spawn_boxes.launch.py world:=<월드명>
ign model --list | grep box1            # box1 생성 확인
```
- 좌표·이름을 바꾸려면 `spawn_boxes.launch.py`의 `-x -y -name` 수정.
- 상 난이도(다중 박스): 파일 내 `spawn_box_B` 주석 해제.

## 4. 캡스톤 노드 실연동 기동
```bash
# box_sdf_path = 설치된 SDF 절대경로 (place 재생성에 사용)
BOX_SDF=$(ros2 pkg prefix nd1_capstone)/share/nd1_capstone/models/box1.sdf

ros2 launch nd1_capstone bringup.launch.py sim_mode:=false \
  --ros-args \
  -p node_c_grasp.world_name:=<월드명> \
  -p node_c_grasp.box_model:=box1 \
  -p node_c_grasp.box_sdf_path:=$BOX_SDF
```
> 주의: launch에서 노드별 파라미터를 위 방식으로 못 넘기면, Node C만 별도 실행하며
> `ros2 run nd1_capstone node_c_grasp --ros-args -p world_name:=<월드명> -p box_sdf_path:=$BOX_SDF` 로 지정.

## 5. 미션 실행 & 검증
```bash
python3 ~/ros2_ws/src/.../scripts/smoke_test.py "A구역 박스를 B구역으로 옮겨줘" 60
```
기대: `[C] grasp 결과: 성공`(box1 삭제) → 이동 → `[C] place 결과: 성공`(B에 box1 재생성) → DONE.
`ign model --list`로 box1이 B구역 좌표(2.5,−1.0)에 다시 보이면 성공.

## 6. 자주 막히는 곳 (트리아지 보강)
| 증상 | 원인 | 조치 |
|------|------|------|
| create/remove "false" | 월드명 불일치 | 2단계로 실제 월드명 재확인 |
| place 시 박스 안 생김 | box_sdf_path 미설정 | `$BOX_SDF` 절대경로 전달 |
| "ros_gz_sim create 없음" | 패키지명 차이 | launch에서 `ros_ign_gazebo`로 교체 |
| Nav2 이동 안 됨 | 맵/초기위치 미설정 | RViz에서 2D Pose Estimate |
| 박스가 바닥 뚫고 빠짐 | z 너무 낮음 | spawn `-z 0.1` 이상 확인 |

## 7. 기준 (사실)
- 본 절차는 텔레포트(방식 A) 기준. 물리 파지(방식 B, DetachableJoint)는 팔 URDF + SDF 사전선언 필요(별도).
- 실제 기동·검증은 Gazebo가 있는 컨테이너에서만 가능.
